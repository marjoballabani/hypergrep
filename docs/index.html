The Problem

AI coding agents spend 60-80% of their tokens figuring out where things are. grep returns raw lines. The agent reads files to understand context. Repeats 50+ times per session.

With ripgrep

1 rg "Matcher" -- 376 lines, 10,174 tokens
2 Read 5 files to understand context -- 9,284 tokens
3 rg "impl.*Matcher" -- 1,122 tokens
20,580 tokens

With Hypergrep

1 hypergrep --model -- codebase map, 1,413 tokens
2 --layer 1 --budget 1000 -- signatures + calls, 1,400 tokens
3 --impact Matcher -- blast radius, 1 token
2,814 tokens (87% less)
The agent doesn't need to read files because Hypergrep returns function signatures, call graphs, and impact analysis inline. The search result is the understanding.

How It Works

Six components working together: trigram index for speed, tree-sitter for structure, call graph for relationships.

Query → Trigram Filter → Regex Verify → Tree-sitter Parse → Symbol Expand → Budget Fit → Result

Trigram Index

Extracts 3-character sequences from every file. Filters candidates before regex runs. Zero false negatives guaranteed.

Tree-sitter AST

Parses code into syntax trees for 16 languages. Expands line matches to full function/class bodies. Lazy: only parses files that match.

Call Graph

Tracks who calls what across files. Enables --callers, --callees, and --impact analysis.

Semantic Compression

Three layers: L0 (names, 15 tokens), L1 (signatures + call graph, 80 tokens), L2 (full code, 200+ tokens). Budget fitting selects best results.

Bloom Filter

Answers "does this codebase use Redis?" in 291 nanoseconds. Parses Cargo.toml, package.json, go.mod for real dependency names.

Mental Model

Generates a ~700 token structural summary: directory layout, key abstractions, entry points, hot spots. Load once, skip 20 searches.

Features

Everything an AI agent needs, accessible via CLI.

Text search (replaces grep)

hypergrep "authenticate" src/
hypergrep -c "pattern" src/          # count
hypergrep -l "pattern" src/          # file names only

Structural search (full function bodies)

hypergrep -s "authenticate" src/

# Returns:
# src/auth.rs:1-8 function authenticate
# fn authenticate(user: &str, pass: &str) -> bool {
#     let hashed = hash_password(pass);
#     check_db(user, hashed)
# }

Semantic compression (signatures + call graph)

hypergrep --layer 1 --budget 500 --json "search" src/

# Returns JSON:
# {
#   "name": "search",
#   "signature": "fn search(&self, pattern: &str) -> Result<Vec<Match>>",
#   "calls": ["trigrams_from_regex", "resolve_query"],
#   "called_by": ["search_structural", "test_search_literal"],
#   "tokens": 85
# }

Impact analysis

hypergrep --impact "hash_password" src/

# Impact analysis for 'hash_password' (depth 3):
#   [depth 1] WILL BREAK   src/auth.rs:authenticate
#   [depth 2] MAY BREAK    src/api.rs:login_handler
#   [depth 3] REVIEW        src/main.rs:setup_routes

More commands

hypergrep --callers "symbol" src/     # who calls this?
hypergrep --callees "symbol" src/     # what does this call?
hypergrep --exists "redis" src/       # O(1) technology check
hypergrep --model "" src/             # codebase overview (~700 tokens)
hypergrep --stats "" src/             # index statistics

Install

Pre-built binary (macOS / Linux)

curl -sSfL https://github.com/marjoballabani/hypergrep/releases/latest/download/hypergrep-installer.sh | sh

From source

git clone https://github.com/marjoballabani/hypergrep.git
cd hypergrep
./install.sh

Set up AI agents

Tell your AI tools to use hypergrep. Works with Claude Code, Cursor, Copilot, and Windsurf:

./hypergrep-setup.sh /path/to/your/project

Uninstall

rm -f $(which hypergrep) $(which hypergrep-daemon)
find ~ -name ".hypergrep" -type d -exec rm -rf {} + 2>/dev/null

Research

Hypergrep builds on established CS foundations and introduces novel contributions for the AI agent use case.

Trigram Indexing

Cox (2012). Decompose regex into required 3-char sequences. Intersect posting lists. 196x speedup on Linux kernel.

Sparse N-grams

GitHub Blackbird (2023). Variable-length n-grams weighted by inverse frequency. Avoids common trigram problem at scale.

Tree-sitter

Incremental parser for 40+ languages. Error-tolerant. Produces concrete syntax trees for structural code analysis.

Novel Contributions

Unified text index + code graph. Lazy tree-sitter. Semantic compression with token budgets. Predictive prefetch. Negative indexing.
